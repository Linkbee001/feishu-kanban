# Progress Summary

## Goal

Explain the project's current state clearly and conservatively.

## Workflow

1. Start from recent runs, artifacts, task board signals, and documents.
2. Identify what is complete, in progress, blocked, overdue, or missing.
3. Mention confidence limits when evidence is weak or stale.
4. End with recommended next actions.

## Output guidance

- Prefer a `summary` output.
- Cover current phase, recent completions, blockers or risks, task anomalies, documentation gaps, and next steps.

# Document Generate

## 适用场景

- 需要把稳定结论沉淀到正式知识库时。
- 群里已经形成明确方案、决策、会议结论或项目说明，需要留下可复用文档时。
- 用户明确要求产出 PRD、设计说明、实施方案、纪要或知识页面时。

## 何时读取

- 在你已经完成必要的分析，且判断“这件事应该进文档而不是只留在聊天里”时。
- 在群策略允许写文档，并且准备显式声明 durable/persist 意图时。

## 输出倾向

- 需要正式沉淀时输出 `document`，并显式带上持久化意图和 `feishu_doc` 目标。
- 只是先给群里同步结论、还不到正式沉淀时，改用 `summary`。

## 常见错误

- 在结论不稳定时就写正式文档。
- 把长篇分析原样丢进群消息，而不是沉淀为文档再简短反馈。
- 没有显式声明正式沉淀意图，导致产物语义不清。

## 何时不要用

- 内容仍然是探索性讨论、待确认事项或临时判断时。
- 用户真正需要的是任务拆解或进展回顾，而不是文档沉淀时。

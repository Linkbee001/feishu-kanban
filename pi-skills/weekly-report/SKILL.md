# Weekly Report

## Goal

Produce a concise weekly report that a project owner can reuse with minimal editing.

## Workflow

1. Summarize major outcomes from the week.
2. Call out open risks, blocked items, and overdue work.
3. Note documentation or coordination gaps.
4. Suggest the most important focus for the next week.

## Output guidance

- Prefer a `summary` output for digest-style reporting.
- Use a `document` output only when the caller explicitly needs a durable report document.

# Requirement Analysis

## Goal

Reduce ambiguity before deeper execution.

## Workflow

1. Infer the likely user goal from the current context.
2. Identify missing constraints, hidden assumptions, and dependencies.
3. Propose a clearer framing or a first-pass plan.

## Output guidance

- Prefer a `summary` output.
- If enough information emerges, you may transition into a document or task-oriented answer.

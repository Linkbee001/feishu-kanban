## Architecture Changes

**Before (v1.0.0):**
- PiMonoAdapter: 2834 lines (monolithic)
- All functionality in single file
- Hard to test individual components

**After (v1.1.0):**
- PiMonoAdapter: 2169 lines (coordinator)
- 7 focused services with clear responsibilities
- Layered architecture (8 layers)
- Testable units with DI wiring

**Key Patterns Established:**
1. Coordinator/Facade pattern (PiMonoAdapter delegates to services)
2. Pure function services (PiPromptBuilder, PiOutputProcessor)
3. Session state layering (in-memory PiSessionStateService + persisted SessionStateService)
4. Tool registry centralization (PiToolRegistry owns all 11 custom tools)

    Per D-17 and D-11: Rewrite useMessages to send search to backend and use standard page-based pagination.

    Replace the current useMessages hook implementation entirely. The new hook must:

    **Interface changes:**
    - Add `search` to `UseMessagesParams` and to the `filters` object in `UseMessagesReturn`
    - Remove `loadMore` and `hasMore` from `UseMessagesReturn`
    - Add `setPage` and `setLimit` to `UseMessagesReturn` for standard pagination control

    **New UseMessagesReturn interface:**
    ```typescript
    interface UseMessagesReturn {
      messages: MessageListItem[];
      total: number;
      page: number;
      limit: number;
      filters: {
        search: string;
        group: string;
        startDate: string;
        endDate: string;
        type: MessageType;
      };
      loading: boolean;
      error: Error | null;
      setFilters: (filters: Partial<UseMessagesReturn['filters']>) => void;
      setPage: (page: number) => void;
      setLimit: (limit: number) => void;
      refetch: () => void;
    }
    ```

    **Implementation changes:**
    1. Use `apiGet` from `useApi.ts` instead of raw `fetch` (consistent with useGroups pattern)
    2. Add `search` to the `queryParams` sent to the API: `if (filters.search) queryParams.set('search', filters.search)`
    3. Remove the `loadMore` / append pattern. When page changes, fetch that page only (replace, not append): `setMessages(data.items)` always, not `[...prev, ...data.items]`
    4. Remove the separate `useEffect` for `page > 1` — use a single `useEffect` that triggers on any filter or page change
    5. When any filter changes, reset page to 1
    6. Default `limit` to 20 (matching DataTable standard)

    The fetchMessages function should be:
    ```typescript
    const fetchMessages = useCallback(async () => {
      setLoading(true);
      setError(null);
      try {
        const queryParams = new URLSearchParams();
        if (filters.search) queryParams.set('search', filters.search);
        if (filters.group) queryParams.set('group', filters.group);
        if (filters.startDate) queryParams.set('startDate', filters.startDate);
        if (filters.endDate) queryParams.set('endDate', filters.endDate);
        if (filters.type !== 'all') queryParams.set('type', filters.type);
        queryParams.set('page', String(page));
        queryParams.set('limit', String(limit));

        const data = await apiGet<MessagesResponse>(`/admin/messages?${queryParams.toString()}`);
        setMessages(data.items);
        setTotal(data.total);
      } catch (err) {
        setError(err instanceof Error ? err : new Error('Failed to fetch messages'));
      } finally {
        setLoading(false);
      }
    }, [filters, page, limit]);
    ```

    Single useEffect: `useEffect(() => { fetchMessages(); }, [fetchMessages]);`

    setFilters must reset page to 1:
    ```typescript
    const setFilters = useCallback((newFilters: Partial<UseMessagesReturn['filters']>) => {
      setFiltersState((prev) => ({ ...prev, ...newFilters }));
      setPage(1);
    }, []);
    ```

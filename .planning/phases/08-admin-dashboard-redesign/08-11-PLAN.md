    **admin.service.ts** — In the `listMessages` method:
    1. Add `search?: string` to the options type (around line 535).
    2. Destructure `search` from options (line 541).
    3. After the existing group filter (line 547-548), add search filter logic to `messageWhere`:
       ```typescript
       if (search) {
         messageWhere.OR = [
           { rawText: { contains: search, mode: 'insensitive' } },
           { senderOpenId: { contains: search, mode: 'insensitive' } },
         ];
       }
       ```
       Note: `senderOpenId` is used because the backend looks up `senderName` from `ProjectMemberProfile` AFTER the Prisma query. To search by sender name at the Prisma level, add a second query approach:
       - If search is provided, also find matching `ProjectMemberProfile` records where `displayName` or `groupNickname` contains the search term, collect their `openId` values, and add them to the OR conditions.
       
       Full search filter implementation:
       ```typescript
       if (search) {
         // Find sender openIds matching the search term
         const matchingSenders = await this.prisma.projectMemberProfile.findMany({
           where: {
             OR: [
               { displayName: { contains: search, mode: 'insensitive' } },
               { groupNickname: { contains: search, mode: 'insensitive' } },
             ],
           },
           select: { openId: true },
         });
         const senderOpenIds = matchingSenders.map((s) => s.openId);
         
         messageWhere.OR = [
           { rawText: { contains: search, mode: 'insensitive' } },
           ...(senderOpenIds.length > 0 ? [{ senderOpenId: { in: senderOpenIds } }] : []),
         ];
       }
       ```
    4. This ensures search matches both message content (rawText) and sender names (via profile lookup).

    Also update `frontend/src/types/dashboard.ts` — add `search?: string` to the `MessagesQueryParams` interface.
